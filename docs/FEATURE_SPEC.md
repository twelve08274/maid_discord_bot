# FEATURE_SPEC.md

# Discord育成Bot 機能仕様書

## 1. プロジェクト概要

このBotは、Discord上でユーザーごとのキャラクターを育成できるBotである。
最初のMVPでは、42へのログイン状況をもとにDaily報酬を付与し、EXP・レベル・stamina・affection・moodを変化させることを目的とする。

優先順位は以下の通り。

1. キャラ育成
2. 生活支援
3. 42支援

---

## 2. MVP範囲

最初に完成させる機能は以下。

- `/ping`
- `/register`
- `/status`
- EXP・Level管理
- stamina・affection・mood管理
- `/link42`
- `/checkin`
- 自動Daily
- 立ち絵表示

---

## 3. コマンド仕様

## 3.1 `/ping`

### 目的

Botが正常に起動しているか確認する。

### 入力

なし。

### 処理

Botが応答可能であれば、簡単な応答を返す。

### 出力例

```txt
Pong!
```

---

## 3.2 `/register`

### 目的

DiscordユーザーをBotに登録し、初期ステータスを作成する。

### 入力

なし。

### 処理

1. DiscordユーザーIDを取得する。
2. `users` テーブルに存在するか確認する。
3. 未登録なら初期ステータスを保存する。
4. 登録済みならエラーではなく案内を返す。

### 初期ステータス

```txt
level = 1
exp = 0
stamina = 100
affection = 0
mood = normal
```

### 出力例

```txt
登録しました。今日から育成を始められます。
```

### 登録済みの場合

```txt
すでに登録済みです。/status で現在の状態を確認できます。
```

---

## 3.3 `/status`

### 目的

ユーザーの現在の育成状態を表示する。

### 入力

なし。

### 処理

1. DiscordユーザーIDからユーザー情報を取得する。
2. 未登録なら `/register` を案内する。
3. 登録済みならステータスをEmbedで表示する。
4. moodに応じた立ち絵画像を表示する。

### 表示項目

- level
- exp
- stamina
- affection
- mood
- 最終ログイン日

### 出力形式

Discord Embedを使用する。

### 未登録の場合

```txt
まだ登録されていません。/register で登録してください。
```

---

## 3.4 `/link42`

### 目的

Discordユーザーと42アカウントを連携する。

### 入力

なし。

### 処理

1. Discordユーザーが登録済みか確認する。
2. 42 OAuth認証URLを生成する。
3. ユーザーに認証URLを送る。
4. FastAPI側でOAuth callbackを受け取る。
5. 認証codeからaccess_tokenとrefresh_tokenを取得する。
6. `/v2/me` で42ユーザー情報を取得する。
7. DiscordユーザーIDと42ユーザーIDを紐づけて保存する。

### 保存先

`ft_links` テーブル。

### 備考

- access_tokenには期限があるため、`token_expires_at` を保存する。
- 期限切れの場合はrefresh_tokenで更新する。
- refreshに失敗した場合は再連携を促す。

---

## 3.5 `/checkin`

### 目的

42へのログイン状況を手動で確認し、Daily報酬を付与する。

### 入力

なし。

### 処理

1. Discordユーザーが登録済みか確認する。
2. 42アカウント連携済みか確認する。
3. 42 APIでlocation履歴を取得する。
4. 今日のlocation履歴があるか確認する。
5. 今日まだDailyを受け取っていなければ報酬を付与する。
6. すでに受け取り済みなら案内を返す。
7. 今日ログインしていなければ案内を返す。

### Daily付与条件

```txt
今日のlocation履歴が1件以上ある
かつ
今日まだDailyを受け取っていない
```

### Daily報酬

```txt
EXP +10
stamina 回復
```

## 3.8 `/achievements`

### 目的

ユーザーの実績を表示する。

### 入力

なし。

### 処理

1. 登録済みか確認する。

- Lv.5到達
- Lv.10到達

- 初めての課題並走募集

## 3.9 `/done <課題名>`

### 目的

完了した42課題を登録する。

### 入力

```txt

```

### 処理

1. 登録済みか確認する。
2. 課題名を受け取る。
3. 同じ課題をすでに登録していないか確認する。
4. `tasks` テーブルに保存する。
5. EXPとaffectionを付与する。
6. 実績解除判定を行う。

### 報酬

```txt
EXP +20
affection +5
```

## 3.10 `/who-done <課題名>`


## 4. 育成仕様

## 4.1 EXP

### 基本方針

ユーザーの行動に応じてEXPを付与する。

### 報酬例

```txt
42ログインDaily: EXP +10
/done: EXP +20
/study: EXP +5
/review-message: EXP +5
/food: EXP +3
/weather: EXP +1 または affection +1
```

---

## 4.2 レベルアップ

### 必要EXP

```txt
必要EXP = level * 100
```

### 処理

1. EXPを加算する。
2. 現在レベルの必要EXP以上になったらレベルアップする。
3. levelを1増やす。
4. 余ったEXPは保持する。
5. レベルアップ通知を出す。

### 例

```txt
level = 1
exp = 95
EXP +10
=> level = 2
=> exp = 5
```

---

## 4.3 stamina

### 基本方針

キャラクターの元気度を表す。

### 仕様

- 初期値は100。
- 42ログインDailyで回復する。
- 放置日数に応じて減少する。
- mood判定に使用する。

---

## 4.4 affection

### 基本方針

キャラクターとの親密度を表す。

### 仕様

- 初期値は0。
- コマンド利用時に増加する。
- `/done`、`/study`、`/review-message` などで増加する。

---

## 4.5 mood

### 基本方針

staminaに応じてキャラクターの状態を変える。

### 判定ルール

```txt
stamina >= 70: 元気
stamina >= 40: 普通
stamina < 40: 疲れ気味
stamina < 20: しょんぼり
```

### 注意

実装では、`stamina < 20` を `stamina < 40` より先に判定する。

## 6. 自動Daily仕様

## 6.1 実行間隔

APSchedulerを使い、30分ごとに自動チェックする。

## 6.2 対象ユーザー

以下を満たすユーザー。

```txt
auto_daily_enabled = true
かつ
42アカウント連携済み
```

## 6.3 処理

1. 対象ユーザー一覧を取得する。
2. 各ユーザーの42 location履歴を確認する。
3. 今日ログイン済みか確認する。
4. 今日未付与ならDailyを付与する。
5. 付与後、可能ならDMで通知する。
6. すでに付与済みならスキップする。

## 6.4 注意

42 APIの制限に引っかからないように、ユーザーごとの確認間隔やリクエスト数に注意する。
